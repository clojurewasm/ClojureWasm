# Japanese chat output

ユーザー向けのチャット応答（画面表示されるテキスト）はすべて日本語で書く。
最初のターンから日本語で応答し、ユーザー入力の言語に関わらず日本語を維持する。
これは ClojureWasm プロジェクト方針：「英語をデフォルト、ただしチャット返答と
`docs/ja/NNNN-*.md` は日本語」を確実に履行するための強制設定である。

## 例外（日本語にしないもの）

以下は **英語のままにする**（プロジェクトの公開コードベース方針より）：

- ソースコードの識別子・関数名・型名・キー名・コメント
- ファイルパス、URL、コマンド、ツール出力をそのまま貼る場合
- コミットメッセージ、PR タイトル／本文
- `README.md`、`.dev/`、`.claude/`、`docs/`（`docs/ja/` を除く）配下の文書
- ADR（`.dev/decisions/NNNN-*.md`）
- ログ、エラーメッセージ、診断出力

`docs/ja/NNNN-*.md` 内：本文は日本語、コードブロック内の識別子は英語のまま。

## 文体

- 簡潔に。冗長な前置き・要約・感想を避ける。
- 技術用語（NaN boxing, HAMT, threadlocal 等）はそのままカタカナ／英語表記で
  かまわない（無理に翻訳しない）。
- 箇条書き・コードブロック・テーブルは Markdown のまま使う。
- ユーザーが英語で質問してきても応答は日本語のまま。

## 教材文体ルール（docs/ja/ 章を書くとき）

`docs/ja/NNNN-*.md` の章本文では、過去に校正で繰り返し直したパターンを
最初から避ける。新規章を書くときは下記を満たすかセルフチェックする。

### 用語の統一

- **「本リポ」は使わない。必ず「本リポジトリ」と書く。** ROADMAP §1.4 の
  プロジェクト名方針に揃える。
- v1 との対比節の決まり文句は「**引っ張られずに本リポジトリの理念で
  整理した点：**」（送り仮名「ず」直後の「に」を入れる）。

### 文末

- **体言止め＋句点（`〜する。`／`〜になる。`／`〜が前提。`）の連発を避ける。**
  教材は説明文体なので `〜します。` / `〜になります。` / `〜が前提です。`
  を基本形にする。箇条書きの最終項目だけ体言止めなのは可。
- 否定の事実説明は「〜ない。」ではなく「〜ありません。」「〜しません。」。
  例：×「frame を一切見ない。」 ○「frame を一切見ません。」

### Feynman 課題の様式

- 節冒頭に **「6 歳の自分に説明するつもりで答えてください。」** を置く。
- 各設問は「〜なのか。1 行で。」の形（「〜なの？ 1 行で。」は使わない）。

### 半英訳の口語片を避ける

英語ニュアンスの片仮名・体言止めが日本語的に不自然になっているもの：

| 避ける             | 推奨                                       |
|--------------------|--------------------------------------------|
| 〜のキモ           | 〜の要点／要所                             |
| 〜の正味           | 〜の中身／実装                             |
| 〜のサボリ         | 〜のサボり（送り仮名）／設計の手抜き       |
| 〜の閉じ           | 〜を閉じる章／〜の最終章                   |
| 設計の主役         | 設計の主題                                 |
| 〜という意味。     | 〜という意味です。                         |
| 設計のしるし       | 〜のための地ならし                         |
| 〜の繋ぎ           | 〜のつなぎ                                 |

### 二重受動・誤活用

- ×「折りたためられる」 ○「折りたためる」（可能動詞は二重にしない）
- ×「使いまわしたいから。」 ○「使い回せるようにするためです。」

### v1 対比節の言い回し

「〜していたが、本リポは〜」という途中省略は避け、対比を完結文にする。

- ×「v1 は X していたが、本リポは Y」
- ○「v1 は X していましたが、本リポジトリは Y しています」

### コードブロック内のコメント・識別子

`docs/ja/` の本文は日本語にしても、**コードブロック内の関数名・型名・
変数名・コメントは英語のまま**。Zig や Clojure のサンプルに日本語コメントを
混ぜない。
