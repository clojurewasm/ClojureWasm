# Task 1.10: eval/reader.zig — Parser (Token → Form)

## Zig Tutorial — reader.zig で初登場する文法

### T.1 `error{ A, B, C }` — 明示的エラーセット

```zig
pub const ReadError = error{
    SyntaxError,
    NumberError,
    StringError,
    OutOfMemory,
};
```

01_03 では `const Error = error{ ... }` を定義して `anyerror!Value` で返していた。
ここでは **Reader 固有のエラーセット** を明示定義。呼び出し側は
`ReadError` だけ処理すればよいことが型で保証される。

`anyerror` との違い:
- `anyerror`: すべてのエラーを含む。呼び出し側は具体的なエラーを絞り込めない。
- `error{ A, B }`: 列挙されたエラーのみ。switch で網羅チェックできる。

form.zig の `formatPrStr` は再帰のため `anyerror` が必要だったが、
reader.zig は再帰しても自分のエラーセット内で閉じるため明示定義できる。

### T.2 `errdefer` — エラー時のみ実行される defer

```zig
var items: std.ArrayList(Form) = .empty;
errdefer items.deinit(self.allocator);
// ... items.append(...) が OutOfMemory を返したら → deinit が実行される
return items.toOwnedSlice(self.allocator);
// ↑ 成功時は toOwnedSlice が所有権を移すので deinit は不要
```

01_04 で `defer` を学んだ。`errdefer` は **エラーで抜けた場合のみ** 実行される
特殊な defer:

| キーワード | 実行タイミング               | 主な用途                   |
|-----------|------------------------------|---------------------------|
| `defer`   | スコープ抜け時に**常に**実行  | ロック解放、リソースクリーン |
| `errdefer` | **エラーで抜けた場合のみ**   | 確保済みメモリの後始末      |

**なぜ `defer` ではダメか**:
`toOwnedSlice` は内部バッファの所有権を呼び出し側に移す。
成功時に `deinit` すると、移した先のバッファが無効になる（二重解放）。
`errdefer` なら成功時はスキップされ、エラー時のみ後始末する。

**Zig のリソース管理の定番パターン**:

```zig
const resource = try allocate();
errdefer free(resource);        // 以下の処理がエラーなら解放
// ... resource を使う処理（try で失敗しうる）
return resource;                // 成功: 所有権を呼び出し側に移す
```

### T.3 `catch return` — エラー変換の短縮形

```zig
const val = std.fmt.parseInt(i64, s, 0) catch return error.NumberError;
```

`catch return` は 01_03 の `try` と `catch blk:` の中間的なパターン:
- `try expr`: エラーをそのまま伝播
- `expr catch return error.X`: エラーを**別のエラーに変換**して返す
- `expr catch blk: { ... }`: エラーを捕捉して代替処理

`parseInt` は `std.fmt.ParseIntError` を返すが、Reader は `ReadError` しか
返せないため、`catch return error.NumberError` で変換する。

### T.4 `std.mem.indexOfScalar` — バイト検索

```zig
if (std.mem.indexOfScalar(u8, s, '\\') == null) return s;
```

スライス `s` の中から `'\\'` (バックスラッシュ) を探す。
見つかれば `?usize` のインデックス、なければ `null`。

01_07 では `StringArrayHashMapUnmanaged.get()` が `?*Keyword` を返した。
`indexOfScalar` も同じく Optional を返す。Zig では「見つからないかもしれない」
操作は常に `?T` で表現する。

`std.mem.eql(u8, a, b)` もここで初登場。2 つのスライスの内容比較:

```zig
if (std.mem.eql(u8, txt, "nil")) { ... }
```

C の `strcmp(a, b) == 0` に相当。ポインタ比較ではなく内容比較。

### T.5 `alloc.alloc(T, n)` — スライスのヒープ確保

```zig
const items = self.allocator.alloc(Form, 2) catch return error.OutOfMemory;
items[0] = Form{ .data = .{ .symbol = .{ .name = "quote" } }, ... };
items[1] = inner;
```

01_05 で出た `alloc.create(T)` は単一オブジェクト `*T` を返した。
`alloc.alloc(T, n)` は n 個のスライス `[]T` を返す。

| 操作            | 返り値    | 確保サイズ       | 用途                  |
|----------------|-----------|----------------|-----------------------|
| `create(T)`    | `*T`      | `@sizeOf(T)`   | 単一オブジェクト       |
| `alloc(T, n)`  | `[]T`     | `@sizeOf(T)*n` | 配列 (スライス)        |
| `dupe(T, src)` | `[]T`     | `@sizeOf(T)*n` | 既存スライスのコピー    |

`readQuote` では `alloc(Form, 2)` で 2 要素のスライスを確保し、
`(quote inner)` のリストを構築する。

### T.6 `std.unicode.utf8Encode` — Unicode → UTF-8 変換

```zig
const cp = std.fmt.parseInt(u21, hex, 16) catch return error.StringError;
var utf8_buf: [4]u8 = undefined;
const len = std.unicode.utf8Encode(cp, &utf8_buf) catch return error.StringError;
```

`\u0041` のような Unicode エスケープを処理する。
`u21` は Unicode コードポイント型（最大 0x10FFFF = 21 ビット）。

処理の流れ:
1. `parseInt(u21, "0041", 16)` → `0x41` (= 'A')
2. `utf8Encode(0x41, &buf)` → `buf[0] = 0x41`, 戻り値 `len = 1`
3. `buf[0..len]` を出力バッファに追加

UTF-8 は可変長エンコーディング (1〜4 bytes):

| コードポイント範囲   | UTF-8 バイト数 | 例                        |
|-------------------|----|---------------------------|
| U+0000 〜 U+007F  | 1  | `A` (0x41) → `41`         |
| U+0080 〜 U+07FF  | 2  | `é` (0xE9) → `C3 A9`     |
| U+0800 〜 U+FFFF  | 3  | `あ` (0x3042) → `E3 81 82` |
| U+10000〜 U+10FFFF | 4 | `𝄞` (0x1D11E) → `F0 9D 84 9E` |

`utf8_buf` が `[4]u8` なのは最大 4 バイトに対応するため。

### T.7 `ReadError!?Form` — error union + optional の合成

```zig
pub fn read(self: *Reader) ReadError!?Form {
    const tok = self.nextToken();
    if (tok.kind == .eof) return null;
    return try self.readForm(tok);
}
```

`ReadError!?Form` は **3 つの状態** を持つ型:
1. `error.SyntaxError` 等 — エラー
2. `null` — 正常終了（EOF に到達）
3. `Form` の値 — 読み取り成功

呼び出し側はこの 3 状態を 2 段階で処理する:

```zig
const form = try reader.read() orelse error.SyntaxError;
//           ^^^                ^^^^^^
//           エラーなら伝播      null なら別のエラーに変換
```

- `try`: error union からエラーを伝播し、成功なら `?Form` を得る
- `orelse`: optional から null を処理し、値があれば `Form` を得る

Rust では `Result<Option<Form>, ReadError>` に相当する型で、
パターンマッチが必要になるが、Zig は `try` + `orelse` の
チェーンで読みやすく処理できる。

---

## 概要

Reader は **Token 列 → Form ツリー** の変換を担当する。
3 段パイプラインの最終段:

```
Source text → [Tokenizer] → Token stream → [Reader] → Form tree
```

再帰下降パーサ (recursive descent parser)。`readForm` が各トークンを
ディスパッチし、コレクションは `readDelimited` で再帰的にパースする。

## Reader 構造体

```zig
pub const Reader = struct {
    tokenizer: Tokenizer,      // トークンソース
    source: []const u8,        // 元のソーステキスト（トークンの text 取得に使用）
    allocator: std.mem.Allocator, // コレクション確保用
    peeked: ?Token = null,     // 1 トークン先読みバッファ
    depth: u32 = 0,            // ネスト深度（無限再帰防止）
    max_depth: u32 = 1024,     // 最大ネスト深度
};
```

**先読み (lookahead)**: `peekToken()` で次のトークンを覗き見できる。
Phase 1 では使わないが、Phase 2 の syntax-quote 等で必要になる。

**なぜ `source` フィールドを持つか**: Token は `start` + `len` のオフセットしか
持たない（ゼロコピー設計）。トークンのテキストを取得するには元のソースが必要。
`tok.text(self.source)` でスライスを得る。

## readForm — コアディスパッチ

```zig
fn readForm(self: *Reader, tok: Token) ReadError!Form {
    return switch (tok.kind) {
        .symbol => self.readSymbol(tok),
        .integer => self.readInteger(tok),
        .float => self.readFloat(tok),
        .string => self.readString(tok),
        .keyword => self.readKeyword(tok),
        .lparen => self.readList(tok),
        .lbracket => self.readVector(tok),
        .lbrace => self.readMap(tok),
        .quote => self.readQuote(tok),
        .symbolic => self.readSymbolic(tok),
        .discard => self.readDiscard(tok),
        .rparen, .rbracket, .rbrace => error.SyntaxError,
        .eof => error.SyntaxError,
        .invalid => error.SyntaxError,
    };
}
```

TokenKind → 処理関数の 1:1 マッピング。01_08 で学んだ `union(enum)` の switch と
同じ**網羅性チェック**が働く。TokenKind の全バリアントを処理しないと
コンパイルエラーになる。

不正なトークン（閉じ括弧、EOF、invalid）はすべて SyntaxError。
閉じ括弧を readForm で受け取るのは「対応する開き括弧なしの `)`」なので構文エラー。

## readDelimited — コレクションパース

```zig
fn readDelimited(self: *Reader, closing: TokenKind, open_tok: Token) ReadError![]const Form {
    _ = open_tok;
    self.depth += 1;
    if (self.depth > self.max_depth) return error.SyntaxError;
    defer self.depth -= 1;

    var items: std.ArrayList(Form) = .empty;
    errdefer items.deinit(self.allocator);

    while (true) {
        const tok = self.nextToken();
        if (tok.kind == .eof) return error.SyntaxError;  // 閉じ括弧なし
        if (tok.kind == closing) break;
        const form = try self.readForm(tok);              // ← 再帰
        items.append(self.allocator, form) catch return error.OutOfMemory;
    }
    return items.toOwnedSlice(self.allocator) catch return error.OutOfMemory;
}
```

**パターン**: 開き括弧を読んだ後、閉じ括弧が来るまで `readForm` を繰り返す。
list `()`, vector `[]`, map `{}` すべてこの関数を共有する。引数 `closing` で
終端トークンを指定する（`.rparen`, `.rbracket`, `.rbrace`）。

**深度チェック**: `(((((...))))` のような深いネストで stack overflow しないよう、
`self.depth` をインクリメントして上限チェック。`defer depth -= 1` で
スコープ抜け時にデクリメント。エラーで抜けても正常に抜けても depth が戻る。

**ArrayList のライフサイクル**:

```
.empty → append(...) → append(...) → toOwnedSlice()
                                       ↓
                              呼び出し側が所有する []const Form
```

`toOwnedSlice` は ArrayList の内部バッファの所有権を移す。
ArrayList 自体はもう使えなくなる（01_08 で解説済み）。

## 文字列エスケープ処理

Tokenizer はエスケープを**解釈しない**（`"hello\nworld"` をそのまま返す）。
Reader の `unescapeString` がエスケープを実際の文字に変換:

```
入力: hello\nworld  (12 bytes, '\' と 'n' が別々の文字)
出力: hello↵world  (11 bytes, \n → 改行 1 バイト)
```

**高速パス**: `\` を含まない文字列は unescapeString が即座にソーススライスを
返す（コピーなし）。`\` を含む場合のみ ArrayList で新しいバッファを構築。

```zig
if (std.mem.indexOfScalar(u8, s, '\\') == null) return s;
//                                              ^^^^^^^^
// バックスラッシュなし → ソースバッファへの参照をそのまま返す
```

この最適化は重要。Clojure コードの大半の文字列にはエスケープがないため、
多くの場合でヒープ確保をスキップできる。

対応エスケープ:

| エスケープ | 文字                   | バイト値 |
|-----------|------------------------|---------|
| `\n`      | 改行                   | 0x0A    |
| `\t`      | タブ                   | 0x09    |
| `\r`      | 復帰                   | 0x0D    |
| `\\`      | バックスラッシュ        | 0x5C    |
| `\"`      | ダブルクォート          | 0x22    |
| `\b`      | バックスペース          | 0x08    |
| `\f`      | フォームフィード        | 0x0C    |
| `\uXXXX`  | Unicode コードポイント  | 可変    |

## quote の展開

```zig
fn readQuote(self: *Reader, tok: Token) ReadError!Form {
    // ...
    const inner = try self.readForm(next);
    const items = self.allocator.alloc(Form, 2) catch return error.OutOfMemory;
    items[0] = Form{ .data = .{ .symbol = .{ .name = "quote" } }, ... };
    items[1] = inner;
    return Form{ .data = .{ .list = items }, ... };
}
```

`'foo` → `(quote foo)` に展開。Reader レベルで展開するのは Clojure の仕様。
Lisp の伝統: quote は「次の式を評価せずにそのまま返す」特殊形式。

`alloc.alloc(Form, 2)` で 2 要素のスライスを確保し、
`items[0]` に quote シンボル、`items[1]` に内側のフォームを配置。
これは `(quote foo)` というリストの Form を構築している。

## シンボルのパース

```zig
fn parseSymbolRef(txt: []const u8) SymbolRef {
    if (std.mem.indexOfScalar(u8, txt, '/')) |idx| {
        if (idx == 0 and txt.len == 1) return .{ .name = txt }; // "/" alone
        return .{ .ns = txt[0..idx], .name = txt[idx + 1..] };
    }
    return .{ .name = txt };
}
```

`/` で分割して namespace と name に分ける。`/` 単独は除算演算子のシンボル。
`nil`, `true`, `false` は Tokenizer が `.symbol` として返すので、
Reader が特別扱いして FormData の `.nil`, `.boolean` に変換する:

```zig
fn readSymbol(self: *Reader, tok: Token) ReadError!Form {
    const txt = tok.text(self.source);
    if (std.mem.eql(u8, txt, "nil")) return Form{ .data = .nil, ... };
    if (std.mem.eql(u8, txt, "true")) return Form{ .data = .{ .boolean = true }, ... };
    if (std.mem.eql(u8, txt, "false")) return Form{ .data = .{ .boolean = false }, ... };
    const sym = parseSymbolRef(txt);
    return Form{ .data = .{ .symbol = sym }, ... };
}
```

**所有権に注意**: `parseSymbolRef` が返す `SymbolRef` の `ns` と `name` は
元のソースバッファへのスライス（借用）。Reader の寿命中のみ有効。
Phase 2 以降で Analyzer が Form → Value 変換するとき、必要に応じて
文字列を `alloc.dupe` でコピーする。

## テストのメモリ管理

```zig
const TestCtx = struct {
    arena: std.heap.ArenaAllocator,

    fn init() TestCtx {
        return .{ .arena = std.heap.ArenaAllocator.init(std.heap.page_allocator) };
    }
    fn deinit(self: *TestCtx) void {
        self.arena.deinit();
    }
    fn readOne(self: *TestCtx, source: []const u8) ReadError!Form {
        var reader = Reader.init(self.arena.allocator(), source);
        return try reader.read() orelse error.SyntaxError;
    }
};
```

`testing.allocator` (GeneralPurposeAllocator) を使うとリーク検出で
コレクションの `[]Form` スライスを個別に free する必要がある。
Reader が返す Form は再帰的にスライスを含む（リストの中にベクタ、等）ため、
全部を正しく free するのは煩雑。

ArenaAllocator なら `deinit()` で一括解放。テストで「構造全体を作って検証して捨てる」
パターンに最適。01_04 で学んだ ArenaGc と同じ原理。
