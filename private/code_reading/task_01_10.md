## Zig Tutorial — reader.zig で初登場する文法

### T.1 `error{ A, B, C }` — 明示的エラーセット

```zig
pub const ReadError = error{
    SyntaxError,
    NumberError,
    StringError,
    OutOfMemory,
};
```

01_03 では `const Error = error{ ... }` を定義して `anyerror!Value` で返していた。
ここでは **Reader 固有のエラーセット** を明示定義。呼び出し側は
`ReadError` だけ処理すればよいことが型で保証される。

`anyerror` との違い:
- `anyerror`: すべてのエラーを含む。呼び出し側は具体的なエラーを絞り込めない。
- `error{ A, B }`: 列挙されたエラーのみ。switch で網羅チェックできる。

form.zig の `formatPrStr` は再帰のため `anyerror` が必要だったが、
reader.zig は再帰しても自分のエラーセット内で閉じるため明示定義できる。

### T.2 `orelse` でエラーを変換する

```zig
pub fn read(self: *Reader) ReadError!?Form {
    const tok = self.nextToken();
    if (tok.kind == .eof) return null;
    return try self.readForm(tok);
}

// 呼び出し側
const form = try reader.read() orelse error.SyntaxError;
//                               ^^^^^^ null を error に変換
```

`reader.read()` は `ReadError!?Form` を返す（エラー or null or Form）。
`try` でエラーを伝播し、`orelse` で null を別のエラーに変換。
3 状態（エラー/null/値）を 2 段階で処理するイディオム。

### T.3 `std.heap.ArenaAllocator` — テストでの一括解放

```zig
const TestCtx = struct {
    arena: std.heap.ArenaAllocator,

    fn init() TestCtx {
        return .{ .arena = std.heap.ArenaAllocator.init(std.heap.page_allocator) };
    }
    fn deinit(self: *TestCtx) void {
        self.arena.deinit();
    }
};
```

Reader はコレクション（`[]const Form`）をヒープに確保する。
`testing.allocator`（GeneralPurposeAllocator）を使うとリーク検出で
全スライスを個別に free する必要がある。

ArenaAllocator なら `deinit()` で一括解放。テストで「構造全体を作って検証して捨てる」
パターンに最適。01_04 で学んだ ArenaGc と同じ原理。

### T.4 `errdefer` — エラー時のみ実行

```zig
var items: std.ArrayList(Form) = .empty;
errdefer items.deinit(self.allocator);
// ... items.append(...) が OutOfMemory を返したら → deinit が実行される
return items.toOwnedSlice(self.allocator);
// ↑ 成功時は toOwnedSlice が所有権を移すので deinit は不要
```

`defer` との違い:
- `defer`: スコープ抜け時に**常に**実行
- `errdefer`: **エラーで抜けた場合のみ**実行

`toOwnedSlice` は内部バッファの所有権を移すため、成功時に `deinit` すると
二重解放になる。`errdefer` ならエラー時のみ解放するので安全。

### T.5 `std.fmt.parseInt` と `std.fmt.parseFloat` — 数値パース

```zig
const val = std.fmt.parseInt(i64, s, 0) catch return error.NumberError;
//                                   ^ base=0: プレフィックスで判定 (0x=hex, 0o=octal, etc.)
```

`parseInt(T, str, base)` は文字列を整数に変換。`base=0` はプレフィックス自動判定:
- `0x...` → 16 進数
- `0o...` → 8 進数
- `0b...` → 2 進数
- それ以外 → 10 進数

`catch return error.NumberError` でパースエラーを Reader のエラーに変換。

## 概要

Reader は **Token 列 → Form ツリー** の変換を担当する。
3 段パイプラインの最終段:

```
Source text → [Tokenizer] → Token stream → [Reader] → Form tree
```

再帰下降パーサ (recursive descent parser)。`readForm` が各トークンを
ディスパッチし、コレクションは `readDelimited` で再帰的にパースする。

## 構造

```zig
pub const Reader = struct {
    tokenizer: Tokenizer,      // トークンソース
    source: []const u8,        // 元のソーステキスト
    allocator: std.mem.Allocator, // コレクション確保用
    peeked: ?Token = null,     // 1 トークン先読みバッファ
    depth: u32 = 0,            // ネスト深度（無限再帰防止）
    max_depth: u32 = 1024,     // 最大ネスト深度
};
```

**先読み (lookahead)**: `peekToken()` で次のトークンを覗き見できる。
Phase 1 では使わないが、Phase 2 の syntax-quote 等で必要になる。

## readForm — コアディスパッチ

```zig
fn readForm(self: *Reader, tok: Token) ReadError!Form {
    return switch (tok.kind) {
        .symbol => self.readSymbol(tok),
        .integer => self.readInteger(tok),
        .float => self.readFloat(tok),
        .string => self.readString(tok),
        .keyword => self.readKeyword(tok),
        .lparen => self.readList(tok),
        .lbracket => self.readVector(tok),
        .lbrace => self.readMap(tok),
        .quote => self.readQuote(tok),
        .symbolic => self.readSymbolic(tok),
        .discard => self.readDiscard(tok),
        .rparen, .rbracket, .rbrace => error.SyntaxError,
        .eof => error.SyntaxError,
        .invalid => error.SyntaxError,
    };
}
```

TokenKind → 処理関数の 1:1 マッピング。不正なトークン（閉じ括弧、EOF、invalid）
はすべて SyntaxError。

## readDelimited — コレクションパース

```zig
fn readDelimited(self: *Reader, closing: TokenKind, open_tok: Token) ReadError![]const Form {
    self.depth += 1;
    if (self.depth > self.max_depth) return error.SyntaxError;
    defer self.depth -= 1;

    var items: std.ArrayList(Form) = .empty;
    errdefer items.deinit(self.allocator);

    while (true) {
        const tok = self.nextToken();
        if (tok.kind == .eof) return error.SyntaxError;  // 閉じ括弧なし
        if (tok.kind == closing) break;
        const form = try self.readForm(tok);              // ← 再帰
        items.append(self.allocator, form) catch return error.OutOfMemory;
    }
    return items.toOwnedSlice(self.allocator) catch return error.OutOfMemory;
}
```

**パターン**: 開き括弧を読んだ後、閉じ括弧が来るまで `readForm` を繰り返す。
list `()`, vector `[]`, map `{}` すべてこの関数を共有する。

**深度チェック**: `(((((...))))` のような深いネストで stack overflow しないよう、
`self.depth` をインクリメントして上限チェック。`defer depth -= 1` で
スコープ抜け時にデクリメント。

## 文字列エスケープ処理

Tokenizer はエスケープを**解釈しない**（`\"hello\\nworld\"` をそのまま返す）。
Reader の `unescapeString` がエスケープを実際の文字に変換:

```
入力: hello\nworld  (12 bytes)
出力: hello↵world  (11 bytes, \n → 改行)
```

対応エスケープ:
| エスケープ | 文字                   |
|-----------|------------------------|
| `\n`      | 改行 (0x0A)            |
| `\t`      | タブ (0x09)            |
| `\r`      | 復帰 (0x0D)            |
| `\\`      | バックスラッシュ        |
| `\"`      | ダブルクォート          |
| `\b`      | バックスペース (0x08)   |
| `\f`      | フォームフィード (0x0C) |
| `\uXXXX`  | Unicode コードポイント  |

**高速パス**: `\` を含まない文字列は unescapeString が即座にソーススライスを
返す（コピーなし）。`\` を含む場合のみ ArrayList で新しいバッファを構築。

## quote の展開

```zig
fn readQuote(self: *Reader, tok: Token) ReadError!Form {
    const inner = try self.readForm(next);
    const items = self.allocator.alloc(Form, 2);
    items[0] = Form{ .data = .{ .symbol = .{ .name = "quote" } } };
    items[1] = inner;
    return Form{ .data = .{ .list = items } };
}
```

`'foo` → `(quote foo)` に展開。Reader レベルで展開するのは Clojure の仕様。
Lisp の伝統: quote は「次の式を評価せずにそのまま返す」特殊形式。

## シンボルのパース

```zig
fn parseSymbolRef(txt: []const u8) SymbolRef {
    if (std.mem.indexOfScalar(u8, txt, '/')) |idx| {
        if (idx == 0 and txt.len == 1) return .{ .name = txt }; // "/" alone
        return .{ .ns = txt[0..idx], .name = txt[idx + 1..] };
    }
    return .{ .name = txt };
}
```

`/` で分割して namespace と name に分ける。`/` 単独は除算演算子のシンボル。
`nil`, `true`, `false` は Tokenizer が `.symbol` として返すので、
Reader が特別扱いして FormData の `.nil`, `.boolean` に変換する。
