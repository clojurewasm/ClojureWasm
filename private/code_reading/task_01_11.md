# Task 1.11: main.zig — Minimal CLI

## Zig Tutorial — main.zig で初登場する文法

### T.1 `std.process.args()` — コマンドライン引数

```zig
var args = std.process.args();
_ = args.skip(); // プログラム名をスキップ

while (args.next()) |arg| {
    // arg: []const u8 — 各引数の文字列
}
```

`std.process.args()` はイテレータを返す。C の `argv` に相当するが、
`argc` を持たずイテレータパターンで消費する。

`.skip()` は次の要素を読み捨てる（返り値は `?[]const u8` だが `_` で破棄）。
プログラム名（`argv[0]`）は通常不要なので最初にスキップ。

### T.2 `std.fs.File.stdout()` — 標準出力

```zig
var stdout_buffer: [4096]u8 = undefined;
var stdout_writer = std.fs.File.stdout().writer(&stdout_buffer);
const stdout = &stdout_writer.interface;
```

Zig 0.15 の I/O は **バッファ付き Writer** が標準。
`.writer(&buf)` でバッファリングされた writer を取得し、
`.interface` で `Io.Writer` インターフェースを得る。

`try stdout.flush()` でバッファの内容を実際に書き出す。
`print` だけではバッファに溜まるだけで画面に表示されない場合がある。

### T.3 `std.process.exit(1)` — プロセス終了

```zig
std.process.exit(1);
```

C の `exit(1)` に相当。ステータスコード 1 で即座にプロセスを終了。
Zig の `main` は `!void` を返すので通常は `return error.X` でエラー終了するが、
CLI ツールではユーザ向けメッセージを出してから `exit` する方が親切。

### T.4 `@errorName(err)` — エラー名の文字列取得

```zig
reader.read() catch |err| {
    try stderr.print("Read error: {s}\n", .{@errorName(err)});
```

`@errorName(err)` はエラー値を `[]const u8` 文字列に変換するビルトイン。
例: `error.SyntaxError` → `"SyntaxError"`。
デバッグ出力やユーザ向けエラーメッセージに便利。

## 概要

Phase 1 の最終タスク（gate を除く）。Reader パイプラインの動作確認用 CLI。

```
$ cljw -e '(+ 1 2)'
(+ 1 2)
```

「読んで表示する」だけ（eval なし）。Reader の round-trip 検証:
ソーステキスト → Tokenizer → Reader → Form → toString → 出力。

## CLI 構造

```
cljw                    → "ClojureWasm" を表示
cljw -e '(+ 1 2)'      → 式を読んで表示
cljw -h / --help        → ヘルプ表示
cljw --eval '42'        → -e の長い形式
```

引数パースは `std.process.args()` イテレータをそのまま while で回すだけ。
Phase 14 で本格的な CLI（ファイル読み込み、REPL 等）に拡張する。

## メモリ管理

```zig
var arena = std.heap.ArenaAllocator.init(std.heap.page_allocator);
defer arena.deinit();
const alloc = arena.allocator();

var reader = Reader.init(alloc, source);
```

ArenaAllocator を使い、プロセス終了時に一括解放。
短命なCLI 実行では個別 free のオーバーヘッドを避ける。
Reader が内部でコレクションの `[]Form` を確保するので、
アロケータは必須。

## Form → 文字列の経路

`formatPrStr` は `std.io.AnyWriter` を受け取るが、
`main.zig` の stdout は Zig 0.15 の新しい `Io.Writer` 型。
互換性のため `toString()` で文字列に変換してから print する:

```zig
const s = form.?.toString(alloc) catch |err| { ... };
try stdout.print("{s}\n", .{s});
```

Phase 2 以降で `Io.Writer` → `AnyWriter` のブリッジを整備すれば
直接ストリーミング出力に切り替えられる。
